# Describe the robot once.
emb = sv.embodiments.register(
    templates.so101())

# Bind a policy: model + embodiment.
policy = sv.policy("pi0.5", emb,
    instruction="pick up the cube")

# The control loop is a Session.
with sv.session(policy) as sess:
    for obs in robot.stream():
        robot.execute(sess.act(obs))

Run, fine-tune, and deploy from one API

Every model has a different interface. Servo gives you one.

RUN

One API across every model

Call any policy or world model through one typed contract. Preprocessing, normalization, and action decoding handled for you.

TRAIN & EVALUATE

Fine-tune, then prove it

Fine-tune a base policy on your LeRobot dataset, then gate every version in simulation before it ships.

DEPLOY

Cloud or on the robot

Serve from a hosted endpoint, or export an optimized build that runs the real-time loop on the robot.

Examples

01Import a dataset from the Hub
ds = sv.datasets.from_hf(
  "acme/so101-pick-place",
  embodiment=emb)
02Fine-tune on your data
variant = ds.finetune(
  base="smolvla",
  method="lora").wait().variant
03Evaluate in simulation
variant.evaluate(
  rung="sim_benchmark",
  benchmark="libero-90").wait()
04Deploy, then run a Session
dep = variant.deploy(embodiment=emb)
with sv.session(dep.policy()) as s:
  robot.execute(s.act(obs))

Two execution surfaces: actions and rollouts

A Policy returns actions; a World returns rollouts. Authenticate with a bearer token; the request names the model and your embodiment. For the real-robot control loop, open a Session over /v1/realrobot/sessions.

POST /v1/action-predictionsVLA policy

  -H "Authorization: Bearer $SERVO_KEY" \

Deploy to the cloud or the robot

Hosted endpoints for everything async. Edge export for the real-time control loop.

Hosted endpoint

async · batch · latency-tolerant

  • World-model rollouts and synthetic data generation
  • Offline and simulation evaluation at scale
  • Teleop assist and remote control with action-chunking
  • Usage telemetry per org (frames, GPU-seconds), scales to zero

Edge export

real-time · safety-critical · on-robot

  • Optimized and quantized for Jetson and Thor-class hardware
  • Packaged with its runtime, with versioned updates
  • Runs the 10–50 Hz control loop locally
  • No cloud round-trip in the control path
